# S3 Scenario Fields

S3 scenarios provide a small S3-compatible path-style object API backed by
in-memory storage. This is useful for tests that need object upload, download,
delete, and bucket listing behavior without running MinIO or AWS S3.

Supported routes:

```text
PUT    /s3/:bucket/:key
GET    /s3/:bucket/:key
HEAD   /s3/:bucket/:key
DELETE /s3/:bucket/:key
GET    /s3/:bucket
```

Example:

```yaml
path: "/s3/:bucket/:key"
method: GET
type: s3
description: "S3-compatible GET object mock"
scenarios:
  - name: "get object"
    contentType: "application/json"
    body: |
      {
        "message": "default object body"
      }
```

Top-level fields:

| Field | Required | Description |
| --- | --- | --- |
| `path` | yes | HTTP route used for path-style S3 matching. |
| `method` | yes | `GET`, `HEAD`, `PUT`, or `DELETE`. |
| `type` | yes | Must be `s3`. |
| `description` | no | Text shown in the control UI. |
| `scenarios` | yes | Ordered list of S3 scenarios. |

Scenario fields:

| Field | Required | Description |
| --- | --- | --- |
| `name` | no | Human-readable scenario name shown in the UI. |
| `bucket` | no | Bucket override. Defaults to the bucket in the request path. |
| `key` | no | Object key override. Defaults to the key in the request path. |
| `dryRun` | no | When `true`, skips object storage and returns a JSON response. |
| `contentType` | no | Default object content type. |
| `body` | no | Default object body returned by `GET` when the object was not previously uploaded. |
| `metadata` | no | Default metadata returned as `x-amz-meta-*` headers. |
| `headers` | no | Extra response headers for object reads. |

Smoke test:

```bash
curl -X PUT http://localhost:7777/s3/demo-bucket/readme.txt \
  -H "Content-Type: text/plain" \
  --data "hello from gigamock"

curl http://localhost:7777/s3/demo-bucket/readme.txt
curl http://localhost:7777/s3/demo-bucket
curl -X DELETE http://localhost:7777/s3/demo-bucket/readme.txt
curl http://localhost:7777/internal/v1/s3/metrics
```

Dry-run check:

```bash
curl http://localhost:7777/s3-dry-run/demo-bucket/readme.txt
```

Runtime metrics:

```bash
curl http://localhost:7777/internal/v1/s3/metrics
```
